Cross-Tenancy Email Issue Troubleshooting

This document explains why adding accounting@D2CustomGears.com to the Outlook profile of accounting@ConseroInc.com is causing issues and outlines short-term and long-term solutions.

The Issue

The root cause of this issue is the cross-tenancy setup between Consero's Microsoft-sold Office 365 tenancy and D2CustomGears' GoDaddy-provisioned Office 365 tenancy. Outlook Desktop does not natively support cross-tenancy authentication when trying to add mailboxes from one tenancy to an Outlook profile logged into another.

Specifically, Outlook requires the user to authenticate against the correct tenancy to access a mailbox. Since the primary account (accounting@ConseroInc.com) is authenticated against Consero's Office 365 tenancy, it cannot directly authenticate to access accounting@D2CustomGears.com, which belongs to the GoDaddy tenancy.

Short-Term Solutions

Option 1: Add as a Separate Account

  1. Open Outlook.
  2. Go to File > Account Settings > Account Settings.
  3. Under the Email tab, click New.
  4. Enter the email address accounting@D2CustomGears.com and authenticate using its credentials.
  5. If successful, the mailbox will appear as a separate account in Outlook.
Note: If Outlook prevents this due to cross-tenancy restrictions, proceed to Option 2.

Option 2: Use a Shared Mailbox Approach

  1. Ensure accounting@ConseroInc.com has delegate or full access permissions to the D2CustomGears mailbox.
  2. In Outlook, go to File > Account Settings > Account Settings.
  3. Select the primary account and click Change.
  4. Click More Settings > Advanced.
  5. Under Open these additional mailboxes, add accounting@D2CustomGears.com.

Option 3: Use Web Access

Access the D2CustomGears mailbox separately via Outlook Web Access (OWA):

  1. Log in at https://login.microsoftonline.com using accounting@D2CustomGears.com.
  2. Use the web portal to send and receive emails.

Mid-Term Solution: Email Forwarding

To simplify access temporarily, set up email forwarding:

  1. Log in to the GoDaddy Office 365 Admin Portal.
  2. Go to the mailbox settings for accounting@D2CustomGears.com.
  3. Set up forwarding to accounting@ConseroInc.com.
Note: Forwarding ensures all emails from D2CustomGears are accessible in the ConseroInc mailbox.

Long-Term Solution: Consolidating Tenancies

The ideal solution is to consolidate all Office 365 accounts into a single Microsoft-managed tenancy. This avoids cross-tenancy authentication issues and simplifies administration.

Benefits

  • Streamlined administration and billing.
  • Seamless mailbox sharing and delegation.
  • Elimination of cross-tenancy complications.

Migration Steps

  1. Plan the migration from the GoDaddy tenancy to the Microsoft tenancy.
  2. Use tools like Microsoft Exchange Admin Center or third-party migration tools.
  3. Retain MX records and DNS entries during migration to avoid downtime.
